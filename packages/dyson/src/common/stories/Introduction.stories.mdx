## Welcome to Dyson

Dyson is Design library at crusher. We use this extensively to avoid rebuilding and rewriting stuff.

The dyson is created to keep in mind.<br/>
1.) Make workflow fast. <br/>
2.) A system to building component and documenting it.<br/>
3.) Consistency across the projects.

## Feature of the system

-   Extendable. We don't like to block people.<br/>
-   Built for speed. <br/>
-   Opinionated. We created this system to make us fast.<br/>
-   We're optimising for speed over accessibility.<br/>

## How we work on this system?

-   See if component exist in the system. If yes, use it.<br/>
-   If it's new component, code it in source and let it evolve. If you think you can abstract out to design library, do that.
    We find wasting too much time on creating component without different context, It can be both limiting and makes us slow.

## Is this ready to use by other team?

Feel free to do so but we can't provide support to any issues or pull request.

Figma style link can be found here https://www.figma.com/file/RdgrDiGxjV4bhOuXg1Jk8K/DLS

We try to follow atomic design and middle out approach to building component.
